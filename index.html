Dúvidas frequentes

Perguntas frequentes

Qual o prazo para obter o NIF para estrangeiro? +
O NIF para estrangeiros é geralmente obtido no mesmo dia do atendimento presencial. Com procuração, podemos tratar em nome do cliente sem a sua presença física.
Quais documentos preciso para constituir uma empresa? +
BI/CC do(s) sócio(s), NIF, comprovativo de morada, e o pacto social. Tratamos de todo o processo de A a Z.
É possível fazer atendimento por videochamada? +
Sim! Oferecemos atendimento por videochamada para a maioria dos serviços. Pode marcar através do nosso sistema de marcações online.
Quanto tempo demora uma marcação de automóvel? +
A alteração de propriedade demora tipicamente 15-30 minutos de atendimento. O registo pode levar 2-5 dias úteis para processar.
Posso cancelar ou reagendar a minha marcação? +
Sim, pode cancelar ou reagendar até 2 horas antes do atendimento através do link enviado no email de confirmação, sem qualquer custo.
Trabalham com empresas de todas as dimensões? +
Sim, trabalhamos com freelancers, microempresas, PMEs e grandes empresas. Adaptamos os nossos serviços à realidade de cada cliente.
O atendimento tem algum custo? +
A primeira reunião de diagnóstico é sempre gratuita. Os serviços têm preços fixos transparentes disponíveis na página de cada serviço.
Como recebo a confirmação da marcação? +
Enviamos imediatamente um email de confirmação com todos os detalhes, incluindo referência única, data, hora e documentos a trazer.

-- POSTGRESQL SCHEMA

-- Categorias
CREATE TABLE categories (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug        VARCHAR(80) UNIQUE NOT NULL,
  name        VARCHAR(120) NOT NULL,
  icon        VARCHAR(20),
  color       VARCHAR(20),
  sort_order  INT DEFAULT 0,
  active      BOOLEAN DEFAULT true,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Serviços
CREATE TABLE services (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category_id   UUID REFERENCES categories(id),
  slug          VARCHAR(120) UNIQUE NOT NULL,
  name          VARCHAR(200) NOT NULL,
  description   TEXT,
  duration_min  INT DEFAULT 30,
  price_min     DECIMAL(8,2),
  price_max     DECIMAL(8,2),
  docs_required JSONB,          -- ["BI", "NIF", ...]
  modalities    TEXT[],         -- {presencial, videocall, contacto}
  active        BOOLEAN DEFAULT true,
  created_at    TIMESTAMPTZ DEFAULT now()
);

-- Slots de disponibilidade (templates semanais)
CREATE TABLE availability_templates (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  day_of_week SMALLINT NOT NULL,   -- 1=Mon ... 7=Sun
  start_time  TIME NOT NULL,
  end_time    TIME NOT NULL,
  capacity    SMALLINT DEFAULT 1,
  active      BOOLEAN DEFAULT true
);

-- Marcações
CREATE TABLE bookings (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  reference     VARCHAR(20) UNIQUE NOT NULL,  -- BCE-2025-XXXX
  service_id    UUID REFERENCES services(id),
  modality      VARCHAR(20) NOT NULL,          -- presencial|videocall|contacto
  booking_date  DATE NOT NULL,
  booking_time  TIME NOT NULL,
  status        VARCHAR(20) DEFAULT 'pendente', -- pendente|confirmada|cancelada|concluida
  -- Cliente
  client_name   VARCHAR(200) NOT NULL,
  client_email  VARCHAR(200) NOT NULL,
  client_phone  VARCHAR(30),
  client_nif    VARCHAR(15),
  notes         TEXT,
  -- Metadata
  ip_address    INET,
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now(),
  confirmed_at  TIMESTAMPTZ,
  cancelled_at  TIMESTAMPTZ,
  cancel_reason TEXT
);

-- Índices
CREATE INDEX idx_bookings_date ON bookings(booking_date, booking_time);
CREATE INDEX idx_bookings_status ON bookings(status);
CREATE INDEX idx_bookings_email ON bookings(client_email);
CREATE INDEX idx_bookings_ref ON bookings(reference);
CREATE INDEX idx_services_category ON services(category_id);

-- FAQ
CREATE TABLE faqs (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  question   TEXT NOT NULL,
  answer     TEXT NOT NULL,
  category   VARCHAR(80),
  sort_order INT DEFAULT 0,
  active     BOOLEAN DEFAULT true
);

-- Notificações / Emails
CREATE TABLE notifications (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  booking_id  UUID REFERENCES bookings(id),
  type        VARCHAR(40),    -- confirmation|reminder|cancellation
  channel     VARCHAR(20),    -- email|sms|whatsapp
  sent_at     TIMESTAMPTZ,
  status      VARCHAR(20) DEFAULT 'pending'
);

⚙️ Fluxo do Sistema de Marcações

  • 1. Utilizador abre modal de marcação
  • 2. Seleciona tipo (presencial / videocall / contacto)
  • 3. Filtra e seleciona categoria + serviço
  • 4. API: GET /api/availability?service=X&month=YYYY-MM
  • 5. Calendário mostra dias disponíveis
  • 6. API: GET /api/slots?service=X&date=YYYY-MM-DD
  • 7. Mostra slots livres (exclui já marcados)
  • 8. Utilizador preenche dados pessoais
  • 9. POST /api/bookings — cria registo + gera referência
  • 10. Email de confirmação automático (Resend/SendGrid)
  • 11. SMS opcional (Twilio)
  • 12. WhatsApp opcional (API oficial)
  • 13. Lembrete automático 24h antes

🔔 Regras de Negócio

  • Marcações até 90 dias no futuro
  • Cancelamento até 2h antes grátis
  • Lembrete 24h antes por email/SMS
  • Referência: BCE-YYYY-NNNN
  • Máximo 2 marcações ativas por email
  • Confirmação dupla para videochamadas
  • Tempo de buffer 10 min entre slots
  • Horário: Seg-Sex 09:00–18:00
  • Bloqueio automático de feriados
  • Admin pode forçar slots extra

📡 API Endpoints (Next.js)

  • GET /api/categories
  • GET /api/services?category=X
  • GET /api/availability?service=X&month=M
  • GET /api/slots?service=X&date=D
  • POST /api/bookings
  • GET /api/bookings/[ref]
  • PATCH /api/bookings/[ref]/cancel
  • GET /api/admin/bookings
  • PATCH /api/admin/bookings/[id]
  • GET /api/admin/stats

🔒 Segurança & Compliance

  • Rate limiting: 10 req/min por IP
  • CAPTCHA em formulários públicos
  • Dados encriptados em repouso (AES-256)
  • RGPD: consentimento explícito
  • Retenção de dados: 2 anos
  • Admin auth: NextAuth.js + 2FA
  • Logs de auditoria completos
  • Backup automático diário
